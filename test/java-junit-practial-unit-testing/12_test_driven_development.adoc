(그래서 js 쪽이 describe 로 시작하는건가)

TDD 에서 단위 테스트를 시스템의 모양을 잡고 통제하는 도구로 활용해야 한다

== TDD 의 주된 이익

단위 테스트의 사후 작성: 코드가 예상한 대로 동작한다는 자신감을 얻는 것 +
TDD 로는 동일한 이익 + 그 이상

TDD 를 잘 따른다면 구현하는 실질적인 모든 사례에 대한 단위 테스트를 작성하게 된다
